# CardioAI: End-to-End Heart Disease Prediction & Explainer System

CardioAI is a production-ready machine learning system designed to predict coronary heart disease risk. By analyzing patient clinical attributes, it evaluates risk probability and provides explainable AI insights using SHAP. The system features a modular Python ML pipeline, a FastAPI REST service, and an interactive Streamlit diagnostic dashboard.

> [!IMPORTANT]
> **Disclaimer:** This project is built for educational and demonstration purposes only. Predictions generated by the models must NOT be treated as clinical diagnoses or formal medical advice.

---

## 🛠️ Key Technical Implementations

- **Clinical Preprocessing Pipeline:** Handled missing values (median imputation), removed duplicate records, and implemented IQR-based outlier capping on physiological features (blood pressure, cholesterol, etc.) to minimize variance.
- **Feature Engineering:** Extracted high-impact cardiovascular indicators:
  - *Max Heart Rate Ratio:* (thalach / (220 - age))
  - *Double Product:* (trestbps * thalach) indicating myocardial oxygen demand.
  - *BP-Cholesterol Interaction:* (trestbps * chol) interaction term.
  - *Age Grouping:* Binned cohorts to improve classification boundaries.
- **Model Exploration & Selection:** Implemented and compared 7 classification models (Logistic Regression, Random Forests, XGBoost, Decision Trees, SVM, KNN, and Gradient Boosting).
- **Hyperparameter Tuning:** Conducted GridSearchCV cross-validation on the top-performing model to optimize performance.
- **Model Explainability (SHAP):** Unified SHAP Explainer integration to output local Waterfall plots (explaining individual patient risk factors) and global beeswarm plots.
- **RESTful API Services:** FastAPI backend with Pydantic schema validation and automatic SQLite history logging for audit telemetry.
- **Dashboard UI/UX:** Styled a responsive healthcare blue-and-white theme in Streamlit featuring a multi-tab view, CSV batch upload capabilities, interactive Plotly visualizations, and PDF report exporting.

## 📊 Model Evaluation & Metrics

The models were evaluated on an independent test split (20%) stratified by the target label. Below is the final model comparison:

| Model | Accuracy | Precision | Recall | F1-Score | ROC-AUC |
| :--- | :---: | :---: | :---: | :---: | :---: |
| **Logistic Regression** | **86.88%** | **81.25%** | **92.86%** | **86.67%** | **94.91%** |
| **Gradient Boosting** | 86.88% | 83.33% | 89.28% | 86.20% | 95.02% |
| **Random Forest** | 86.88% | 83.33% | 89.28% | 86.20% | 94.21% |
| **XGBoost** | 85.24% | 80.64% | 89.28% | 84.74% | 94.37% |
| **Support Vector Machine** | 81.96% | 77.41% | 85.71% | 81.35% | 91.88% |
| **KNN** | 75.40% | 70.96% | 78.57% | 74.57% | 83.87% |
| **Decision Tree** | 70.49% | 64.70% | 78.57% | 70.96% | 71.10% |

*Note: Logistic Regression was selected as the final production model due to its high F1-score, sensitivity (recall), and excellent generalization.*

## 🚀 Getting Started & Setup

### Prerequisites
- Python 3.12+
- Docker & Docker Compose (Optional)

### Installation
1. Clone the repository:
   ```bash
   git clone https://github.com/adil-hussa1n/heart-disease-ai.git
   cd heart-disease-ai
   ```

2. Setup virtual environment:
   ```bash
   python -m venv .venv
   # Windows:
   .venv\Scripts\activate
   # Linux/macOS:
   source .venv/bin/activate
   ```

3. Install requirements:
   ```bash
   pip install -r requirements.txt
   ```

---

## 🏃 Pipelines and Launch Instructions

### 1. Preprocessing & Feature Engineering
Fetches the raw UCI Cleveland dataset and runs the cleaning/scaling pipeline:
```bash
python fetch_data.py
python -m src.preprocessing
```

### 2. Model Training & Comparison
Runs cross-model comparisons, tunes the hyperparameter grids, and generates evaluation files in `reports/`:
```bash
python -m src.train
```

### 3. FastAPI REST Server
Launch FastAPI server locally:
```bash
uvicorn api.main:app --reload --port 8000
```
Interactive Swagger API documentation is available at [http://127.0.0.1:8000/docs](http://127.0.0.1:8000/docs).

### 4. Streamlit Dashboard
Launch Streamlit app locally:
```bash
streamlit run app/streamlit_app.py
```
Open [http://localhost:8501](http://localhost:8501) in your browser.

## 🐳 Container Deployment

To run both services (FastAPI and Streamlit) inside containerized environments:
```bash
docker-compose up --build
```

- FastAPI Docs: [http://localhost:8000/docs](http://localhost:8000/docs)
